# 有声读物播放器 - 需求与实现计划

## 第一部分：需求文档

### 1. 项目概述

本项目是一个面向家庭使用的有声读物播放器系统，运行在家庭服务器（如NAS）上。系统包含用户播放界面和管理员后台界面，支持多用户管理、播放时长限制、播放统计等功能。

### 2. 用户角色

#### 2.1 普通用户
- **目标用户**：儿童、家人
- **访问方式**：通过浏览器访问网页应用
- **认证方式**：简单用户名/密码登录
- **权限**：只能播放管理员预先选定的内容

#### 2.2 管理员
- **数量**：单一管理员
- **访问方式**：独立的后台管理页面
- **认证方式**：管理员专用登录页面
- **权限**：管理所有系统配置和内容

### 3. 功能需求

#### 3.1 用户播放界面

##### 3.1.1 内容浏览
- 浏览管理员分配的有声读物内容
- 支持按作者/系列/书籍的目录结构浏览
- 显示书籍封面、标题、作者等信息
- 支持搜索和筛选功能

##### 3.1.2 播放控制
- 播放/暂停/停止控制
- 进度条拖动定位
- 音量控制
- 播放速度调节
- 上一首/下一首切换
- 播放列表管理
- 播放进度记忆（续播功能）

##### 3.1.3 时长限制显示
- 显示今日剩余播放时长
- 时长即将耗尽时的提醒
- 达到时长限制时的友好提示

##### 3.1.4 个人统计查看
- 查看个人播放历史
- 查看今日/本周/本月/本年播放统计
- 图表化展示播放数据

#### 3.2 管理员后台界面

##### 3.2.1 内容管理
- 扫描和索引NAS上的有声读物文件
- 支持WebDAV和本地挂载两种访问方式
- 管理有声读物元数据（标题、作者、封面等）
- 按文件夹组织内容
- 批量导入/导出功能

##### 3.2.2 用户管理
- 创建/编辑/删除用户账户
- 重置用户密码
- 启用/禁用用户账户
- 查看用户播放历史

##### 3.2.3 权限配置
- 为每个用户分配可访问的内容文件夹
- 设置全局默认播放时长限制
- 为特定用户设置个性化时长限制
- 支持按用户独立设置和全局统一设置

##### 3.2.4 时长限制管理
- 设置每日播放时长限制（全局和按用户）
- 设置每周播放时长限制（可选）
- 设置每月播放时长限制（可选）
- 临时调整用户时长（如奖励额外时长）

##### 3.2.5 统计报表
- 每日播放时长统计
- 每周播放时长统计
- 每月播放时长统计
- 每年播放时长统计
- 按内容分类统计
- 按用户统计
- 交互式图表展示
- 数据导出功能

### 4. 技术需求

#### 4.1 部署环境
- **运行环境**：家庭服务器（NAS）
- **操作系统**：Linux（Docker容器化部署）
- **网络访问**：局域网内访问

#### 4.2 内容存储
- **存储位置**：NAS上的有声读物文件
- **访问协议**：
  - WebDAV（远程访问）
  - 本地挂载（SMB/CIFS、NFS）
- **文件格式**：MP3、M4A、FLAC等常见音频格式
- **目录结构**：按作者/系列/书籍组织

#### 4.3 技术栈
- **后端**：Python（FastAPI）
- **前端**：React + TypeScript
- **数据库**：PostgreSQL（生产）/ SQLite（开发）
- **部署**：Docker + Docker Compose
- **认证**：JWT with refresh tokens
- **存储**：抽象层支持WebDAV + 本地挂载

#### 4.4 性能要求
- 支持多个用户同时播放
- 音频流媒体传输（支持断点续传）
- 响应式设计，支持各种设备访问

### 7. 约束条件

#### 7.1 技术约束
- 必须运行在家庭服务器环境
- 必须支持网络文件系统访问
- 必须是Web应用（浏览器访问）

#### 7.2 业务约束
- 只能播放管理员预先配置的内容
- 必须严格执行时长限制
- 必须提供详细的播放统计

#### 7.3 性能约束
- 支持至少10个并发用户
- 音频流媒体延迟小于2秒
- 页面加载时间小于3秒

#### 10.1 技术风险
- **风险**：音频格式兼容性问题
- **缓解**：使用FFmpeg进行格式转换

- **风险**：网络文件系统性能问题
- **缓解**：实现本地缓存机制

#### 10.2 业务风险
- **风险**：用户绕过时长限制

- **风险**：内容版权问题
- **缓解**：仅播放用户自有内容，不提供下载功能

## 第二部分：实现计划

### 1. 技术栈确认

**已确认的技术选型：**
- **后端框架**：FastAPI (Python) - 轻量级、高性能的REST API框架
- **前端框架**：React + TypeScript - 成熟的生态系统，适合复杂UI
- **数据库**：PostgreSQL（生产环境）/ SQLite（开发环境）
- **认证方式**：JWT with refresh tokens - 无状态认证
- **存储层**：抽象层支持WebDAV + 本地挂载（SMB/CIFS、NFS）
- **部署方式**：Docker + Docker Compose
- **音频播放**：HTML5 Audio API，支持流媒体传输
- **状态管理**：React Context + useReducer（轻量级）
- **图表库**：Chart.js 或 ECharts（支持交互式图表）
- **测试框架**：pytest + httpx (后端)，Playwright (前端E2E)
- **数据库ORM**：SQLAlchemy
- **数据库迁移**：Alembic

### 8. 风险与缓解

#### 8.1 技术风险
- **存储可靠性**：WebDAV vs 本地挂载在NAS设备上可能不稳定
  - **缓解**：实现健壮的重试/退避、健康检查和回退；允许手动重新索引

- **大型目录导入**：NAS内容可能很大；导入可能很慢
  - **缓解**：增量导入；分页；有界工作池的后台处理

#### 8.2 安全风险
- **JWT令牌管理和管理员访问**
  - **缓解**：严格的RBAC、令牌轮换、短期访问令牌；安全密码存储

#### 8.3 性能风险
- **大型图表和媒体列表**
  - **缓解**：懒加载、目录分页、长列表虚拟化
